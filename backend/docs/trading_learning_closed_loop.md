# AI交易员学习进化闭环完整文档

**文档整理日期**: 2026年1月28日

---

### 1.1 核心目标

建立完整的交易数据闭环，使AI能够从历史交易中学习，不断优化决策能力：
- 从成功案例中学习盈利模式
- 从失败案例中吸取教训，避免重复错误
- 通过周级进化优化策略参数
- 形成持续改进的交易系统

### 1.2 适用场景

- 盘中实时交易决策
- 每日收盘后复盘学习
- 周度策略参数优化
- 历史交易模式分析

## 三、详细流程说明

### 3.1 交易执行阶段

#### 3.1.1 买入执行流程

**入口函数**: `trading_service.py` - `execute_buy()`

**执行步骤**:
1. **风控检查**:
   - 检查交易时间 (`is_trading_time()`)
   - 检查是否在集合竞价期间
   - 检查是否在午间休市期间
   - 检查是否在收盘集合竞价期间
   - 检查是否在收盘后

2. **参数获取**:
   - 从 `PolicyConfig` 获取进化后的策略参数
   - 获取市场温度，调整乖离率阈值

3. **仓位计算**:
   - 计算单只股票最大仓位（不超过50%）
   - 计算总仓位（不超过90%）
   - 计算可用股数

4. **执行订单**:
   - 调用 `submit_order()` 提交订单
   - 等待订单成交
   - 更新持仓和账户

5. **模式记录**:
   - 在卖出时调用 `_collect_pattern_if_successful()` 记录案例

**关键代码位置**:
- `trading_service.py:447-590` - 买入执行主流程
- `trading_service.py:596-649` - 模式收集逻辑

#### 3.1.2 卖出执行流程

**入口函数**: `trading_service.py` - `execute_sell()`

**执行步骤**:
1. **交易时间检查**:
   - 检查是否在交易时间
   - 检查是否在特殊时段（集合竞价、午间休市等）

2. **获取持仓计划**:
   - 查询 `exit_price=None` 的未卖出计划
   - 计算卖出盈亏比例

3. **执行订单**:
   - 市价单或限价单
   - 更新持仓和账户

4. **模式收集**:
   - 调用 `_collect_pattern_if_successful()` 记录交易案例

**关键代码位置**:
- `trading_service.py:975-1050` - 卖出执行主流程
- `trading_service.py:1031` - 模式收集调用点

#### 3.1.3 模式收集逻辑

**函数**: `_collect_pattern_if_successful(db, plan, pnl_pct)`

**收集内容**:
1. **基本信息**:
   - 股票代码、交易日期、策略名称

2. **市场环境**:
   - 获取买入日的市场快照
   - 包含市场温度、涨停数量等

3. **K线形态**:
   - 获取买入前10天的K线数据
   - 用于分析形态特征

4. **盈亏信息**:
   - 实际盈亏比例
   - 持仓天数
   - 成功标记（盈利≥0.5%为成功）

**关键代码位置**:
- `trading_service.py:597-649` - 完整收集逻辑

### 3.3 每日学习阶段

**入口函数**: `learning_service.py` - `perform_daily_learning()`

**执行时间**: 每天 20:00 (通过scheduler调度)

**学习任务组成**:
```
每日学习任务
├── Phase 1: 策略统计更新 (_update_strategy_stats_sync)
├── Phase 2: 追踪闭环 (_perform_tracking_update)
└── Phase 3: 记忆生成 (_generate_memories)
```

#### 3.3.1 Phase 1: 策略统计更新

**函数**: `_update_strategy_stats_sync()`

**执行内容**:
1. 遍历所有策略
2. 统计近期交易表现:
   - 胜率计算
   - 平均盈亏
   - 最大回撤
3. 更新 `StrategyStats` 表

**关键代码位置**:
- `learning_service.py:460-500` - 策略统计更新

#### 3.3.2 Phase 2: 追踪闭环

**函数**: `_perform_tracking_update()`

**执行内容**:
1. 扫描未执行的交易计划
2. 获取后续T+1~T+5的行情数据
3. 更新追踪数据
4. 判定结果:
   - **Missed(踏空)**: T+3最高涨幅≥6%
   - **RiskAvoided(避险)**: T+3最大跌幅≤-4%
   - **Neutral**: 无显著波动

5. 生成 `OutcomeEvent` 记录

**关键代码位置**:
- `learning_service.py:280-459` - 追踪闭环逻辑

#### 3.3.3 Phase 3: 记忆生成

**函数**: `_generate_memories()`

**执行内容**:
1. **从OutcomeEvent生成反思记忆**:
   - 获取最近3天的Missed/RiskAvoided事件
   - 调用AI生成规则
   - 保存为 `ReflectionMemory`

2. **从失败PatternCase生成反思记忆** (新增):
   - 获取最近3天的失败案例（亏损交易）
   - 调用AI分析失败原因
   - 生成避免重复错误的规则

**关键代码位置**:
- `learning_service.py:48-165` - 记忆生成逻辑

### 3.6 周级进化阶段

**入口函数**: `evolution_service.py` - `run_weekly_evolution()`

**执行时间**: 每周五 19:00 (通过scheduler调度)

**进化操作**:
```
周级进化任务
├── 样本收集 (近30天数据)
├── 影子模式生成 (Evolution)
├── 影子模式评估 (Promotion)
└── 回滚检查 (Rollback)
```

#### 3.6.1 影子模式生成 (Evolution)

**函数**: `_process_strategy_evolution()`

**生成条件**:
1. 样本量≥5个 (动态阈值)
2. 胜率≥55% 或 回撤改善≥20%

**生成内容**:
- 调整策略参数
- 创建新的影子模式配置
- 记录父子版本关系

#### 3.6.2 影子模式评估 (Promotion)

**评估条件**:
- 影子模式运行超过2周
- 样本量≥10个
- 表现优于当前活跃配置

**晋升操作**:
- 将影子模式标记为活跃
- 将旧配置标记为归档

#### 3.6.3 回滚检查 (Rollback)

**回滚条件**:
- 胜率<40%
- 平均回撤>8%

**回滚操作**:
- 标记当前配置为归档
- 恢复上一版本配置

**关键代码位置**:
- `evolution_service.py:75-180` - 进化逻辑

### 3.7 AI决策应用阶段

#### 3.7.1 买入决策中的学习应用

**入口函数**: `analysis_service.py` - `analyze_realtime_trade_signal_v3()`

**应用内容**:
1. **反思记忆** (`_get_relevant_memories()`):
   - 获取该策略相关的反思记忆
   - 标记来源类型 ([失败案例] / [踏空/避险])
   - 作为软约束融入决策

2. **策略统计** (`get_strategy_context()`):
   - 获取策略胜率、平均回撤
   - 根据市场温度调整描述

3. **成功模式** (`get_successful_pattern_context()`):
   - 获取盈利最高的3个案例
   - 参考成功形态和市场环境

4. **失败模式** (`get_failed_pattern_context()`) (新增):
   - 获取亏损最大的3个案例
   - 避免类似错误

**关键代码位置**:

## 五、复盘系统

### 5.1 复盘任务概述

复盘系统是AI交易员的重要组成部分，负责每日收盘后和午间休市期间的市场分析和交易计划生成。

**复盘任务类型**:
- **午间复盘** (11:40): 盘中快速扫描机会
- **收盘复盘** (20:30): 全量数据分析，生成次日交易计划
- **离线学习** (20:00): 执行学习任务，生成反思记忆

### 5.2 午间复盘 (perform_noon_review)

**执行时间**: 每天 11:40

**主要功能**:
1. **市场快照分析**:
   - 获取实时市场数据
   - 分析涨停板梯队
   - 计算市场温度

2. **新闻资讯扫描**:
   - 搜索午间市场新闻
   - 分析板块热点

3. **机会扫描**:
   - 午间强势股扫描
   - 梯队联动机会
   - 成交额Top筛选

4. **交易计划生成**:
   - 生成午间交易计划
   - 最多生成8个计划
   - 最低置信度70%

**关键代码位置**:
- `review_service.py:482-581` - 午间复盘主流程

### 5.3 收盘复盘 (perform_daily_review)

**执行时间**: 每天 20:30 (离线学习完成后)

**主要功能**:
1. **市场数据分析**:
   - 全市场涨跌统计
   - 连板天梯分析
   - 市场温度计算

2. **板块热点分析**:
   - 成交额Top 80
   - 梯队联动机会
   - 趋势过滤

3. **全网资讯搜索**:
   - 宏观新闻
   - 板块利好
   - 政策解读

4. **AI深度复盘**:
   - 多周期分析
   - 选股器追踪
   - 市场总结生成

5. **交易计划生成**:
   - 扫描次日机会
   - 生成买入计划 (最多8个)
   - 生成持仓管理建议

**关键代码位置**:
- `review_service.py:204-402` - 收盘复盘主流程
